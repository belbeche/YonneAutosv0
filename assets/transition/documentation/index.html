Special notes

Running things locally

Due to local security restrictions, some things will not work locally, especially in Chrome.  But everything will work once the files have been updated to your website. I recommend testing in Firefox as is more lenient toward local iframe security. 

To run locally in Chrome you need to start Google Chrome with local file access 

  1. close all instances of Chrome
  2. click Start
  3. type in "run" (without quotes) and hit enter
  4. copy paste this line and hit enter:

chrome --allow-file-access-from-files

Plugin usage

Call the plugin with an options object as an argument. 

$("#container").transitionSlider(options)

 

Creating Slider

Slider options need to have array of slides

$(document).ready(function () {
  $("#container").transitionSlider({
    slides:[
      {src:"assets/1.jpg"},
      {src:"assets/2.jpg"},
      {src:"assets/3.jpg"}
    ]
  })
})

General options

Customizing Slider

You can customize the slider with various plugin options, for example:
$(document).ready(function () {
  $("#container").transitionSlider({
    slides:[
      {src:"assets/1.jpg"},
      {src:"assets/2.jpg"},
      {src:"assets/3.jpg"}
    ],
    width: 590,
    height: 290,
    responsive: false,
    keyboard: {
       enabled: true
    }
  })
})

 

Available options are:

 

options.width [Number]

slider width (if not responsive), used to calculate slider aspect if responsive, default 1000

 

options.height [Number]

slider height (if not responsive), used to calculate slider aspect, default 550

 

options.shadow [String]

slider shadow effect, available "effect1", "effect2", "effect3", "effect4", "effect5", "effect6"

 

options.responsive [Boolean]

fit to parent div (100% width of parent div)

 

options.transitionEffect [String]

global transition effect, can be overridden in each slide, default : "roll", available : "roll", "zoom", "powerzoom", "warp", "stretch", "brightness"

 

options.transitionDuration [Number]

transition duration in ms, default : 1000

 

options.initialSlide [Number]

initial slide number, default : 0

 

options.stopOnLastSlide [Boolean]

stop on last slide (disables loop), default : false

 

options.autoplay [Object]

Object with autoplay parameters, example:

{delay : 1000}
autoplay.delay [Number]

autoplay delay in ms

 

options.pagination [Object]

example: 

{
  style: "effect2",
  dynamicBullets: false,
  clickable: true
}
pagination.style [String]

pagination style name, available "effect1", "effect2", "effect3", "effect4", "effect5"

pagination.dynamicBullets [Boolean]

dynamic bullets, default : true

pagination.clickable [Boolean]

pagination clickable, default : true

 

 

options.navigation [Object]

example: 

{
  style: "effect1"
}
navigation.style [String]

navigation style name, available "effect1", "effect2", "effect3", "effect4", "effect5", "effect6", "effect7", "effect8", "effect9", "effect10"

Slide options

options.slides [Array]

Array of slide objects, example: 

[
  {src:"assets/1.jpg"},
  {src:"assets/2.jpg"},
  {src:"assets/1.mp4"},
...
]
slide.src [String]

slide image or video URL

 

Optional slide parameters:

slide.transitionEffect [String]

name of transition effect

slide.transitionDuration [Number]

transition duration in ms (overrides global transition duration)

slide.direction [String]

transition direction, if not set direction it will be random, possible directions are different for each effect

slide.brightness [String] or [Number]

transition brightness

slide.distance [String] or [Number]

transition distance

slide.easing [String]

transition easing

 

All transition parameters can also be set via transitionEffect property, for example 

transitionEffect: "roll"
direction: "right"
brightness: "fade"

can be shorter written as

transitionEffect: "rollRightFade"

Transition effects

Transition effect can be set globally and for each slide separately. Each effect can be further customized with optional parameters. If direction parameter is not set, random direction will be used. Available effects are

 

Roll

direction : "top", "bottom", "left", "right", "topleft", "topright", "bottomleft", "bottomright"

brightness : "flash", "fade" or Number

distance : "long", "short" or Number

easing : "slow" or "elastic"

 

Stretch

direction : "top", "bottom", "left", "right"

brightness : "flash", "fade" or Number

distance : "long", "short" or Number

easing : "slow" or "elastic"

 

Warp

direction : "top", "bottom", "left", "right"

brightness : "flash", "fade" or Number

distance : "long", "short" or Number

easing : "slow" or "elastic"

 

Zoom

direction : "in", "out"

brightness : "flash", "fade" or Number

easing : "slow" or "elastic"

 

Power zoom

direction : "in", "out"

brightness : "flash", "fade" or Number

easing : "slow" or "elastic"

 

Flash

brightness : Number

easing : "fast", "slow" or "elastic"

 

Fade

brightness : Number

easing : "fast", "slow" or "elastic"

FAQ

Can I load images and videos from another server (cross domain request)?

By default images and videos need to be on the same server. To use resources from another domain you need to add Access-Control-Allow-Origin header to server response, for example to allow http://foo.example to load the resources you need to add header

Access-Control-Allow-Origin: http://foo.example

Slider is not working

Please check your browser console for errors. In most cases the problem is that slider has no slides specified, or is trying to load resources from another domain. There might also be javascript errors on the page caused by another plugins that can break the javascript execution on the page

 

Mixed content error in browser console

If you get error that looks like this

Mixed Content: The page at 'https://www....' was loaded over HTTPS, but requested an insecure XMLHttpRequest endpoint 'http://www....'. This request has been blocked; the content must be served over HTTPS.

that means that your page is serverd over HTTPS so you need to load resources over HTTPS also, so you need to update image and video urls and replace http:// with https://
